# Android App Analyzer v0.2.0 Release Notes

**Release date:** 2026-03-10

## Summary

This release hardens the Android App Analyzer skill for repeated package-based usage, fixes real APKCombo download compatibility issues, and adds Claude Code marketplace metadata. It also aligns the npm package name with the GitHub repository name.

## Included Commits

- `258b405` `docs: add workspace hardening design and plan`
- `5ea2d06` `feat: harden analyzer workspace and download flow`
- `f8d7569` `fix: support direct apkcombo variant links`
- `8d2f166` `feat: add claude marketplace metadata`

## Highlights

### Managed workspace cache

- Added a shared workspace manager for package-based analysis.
- Default cache root is `.cache/android-app-analyzer/`.
- Each package gets its own managed subdirectory.
- Cache thresholds are now enforced:
  - more than 5 package workspaces: warn and continue
  - more than 20 package workspaces: stop and require manual cleanup

### More reliable APKCombo downloads

- Refactored the downloader to support both observed APKCombo flows:
  - direct `a.variant` links with `/checkin` token injection
  - `xid -> /dl` variant listing flow
- Verified against the real package `holy.bible.kjv.kingjamesbible.verse.biblia`.

### Analyzer workflow cleanup

- Moved analyzer outputs into managed workspace directories.
- Added `run.json` metadata output for each analysis run.
- Standardized Python comments, CLI help text, and skill-facing messages in English.
- Made dependency loading lazy so `--help` does not trigger unnecessary installs.

### Claude Code marketplace support

- Added `.claude-plugin/marketplace.json`.
- Documented Claude Code marketplace installation in both English and Chinese README files.

### Naming alignment

- Renamed the npm package from `codex-skill-android-app-analyzer` to `skill-android-app-analyzer`.
- This aligns the npm package name with the GitHub repository name.
- This is a new npm package identity. The previously published package name remains separate.
